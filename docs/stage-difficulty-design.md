# LoopGame ステージ難易度設計書

Stage Difficulty Design Document

## 1. 目的とスコープ

本書は LoopGame のステージ難易度カーブを再設計するための提案書である。
現行実装が持つ「蝶の種別」— サイズ 3 種・スペシャル蝶・色替え蝶・倍率蝶 — と
「お邪魔オブジェクト」3 種(いもむし・ハチ・クモ)を**最大限に活かし**、
各ステージが固有の「学び」と「戦略」を持つように**レベル 1〜20** の構成と、
それ以降(エクストラ帯)の方針を定義する。

-   主対象は `src/scripts/utils/stage-config.json` の設定値。
-   **前提**: お邪魔オブジェクトは `feature/obstacle-objects` ブランチで実装中であり、
    本設計はその**マージを前提**とする。
-   ステージ構成自体は config 変更で実現できるが、ルール伝達に関する演出面の
    依存事項(§4.3)と、レベル 21 以降の生成ロジック変更(§5.3)は
    別途実装課題として切り出す。

## 2. 設計の前提: 現行メカニクスの整理

### 2.1 蝶の種別

| 種別    | 速度 (x/y) | 表示幅(px) | 特徴                                                                                                                               |
| ------- | ---------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| small   | 0.6 / 0.6  | ~31        | 最速。動きが読みにくく、囲うのに先読みが要る                                                                                       |
| medium  | 0.5 / 0.4  | ~41        | 標準。バランス型                                                                                                                   |
| large   | 0.4 / 0.3  | ~51        | 最遅。ただし**体が大きい**ため、ループに収めるには大きな円が要り、捕獲判定円も大きい                                               |
| special | 0.6 / 0.5  | ~74        | レベル 5 の倍数で開始 10 秒後に 1 匹だけ出現。金色に発光。捕獲判定が緩い (hitRate 0.4)。**捕獲してクリアするとボーナスステージへ** |

蝶は**引いた線が体に触れると反転して逃げる**。この線感知はサイズによらず
共通の挙動で、感知半径(hitAreaSize: large 14 / medium 11 / small 9)は
体の半径の約 55% と、どのサイズでもほぼ同じ比率である。つまり
「large だけが線を遠くから察知する」ような種別差は**ない**。
この性質は「線=柵」として蝶を誘導するテクニックの基盤になる。

サイズとは独立に、次の 2 つの「種別修飾」がある。

-   **色替え蝶** (`isButterflyColorChange`): メイン色と別のサブ色を持つ。
    **1 匹だけをループで囲むと色がサブ色に切り替わる**(捕獲されない)。
-   **倍率蝶** (`multipleButterflyRate` / `maxMultipleRate`): 葉っぱに「x2」「x3」が
    表示され、含めて捕獲すると**そのループの得点全体が乗算される**。
    複数含めれば乗算が重なる(x2 と x3 なら ×6)。

### 2.2 捕獲ルール

1 回のループで囲んだ蝶の内訳により結果が分岐する。

| 囲んだ内容                          | 結果                                                       |
| ----------------------------------- | ---------------------------------------------------------- |
| 0 匹                                | 何も起きない(花のみ取得可)                                 |
| 1 匹                                | 捕獲ではなく**色替え**(色替え蝶でない場合は見た目変化なし) |
| 2 匹以上・全員同色                  | 捕獲成功。**匹数 × 10 点**                                 |
| 3 匹以上・**全員バラバラの色**      | 捕獲成功。**匹数 × 20 点**                                 |
| 上記以外(混色で重複あり、異色 2 匹) | **失敗。−20 点**                                           |

-   線は引いてから **1.5 秒**で消える(ロングループ中は 2.0 秒)。
-   捕獲した分だけ新しい蝶が即座に補充される(スペシャル蝶は補充されない)。
-   `needCount` に到達した瞬間にステージ終了。**超過分は 1 匹 100 点のボーナス**
    になるため、「最後のループで大量捕獲して超過を稼ぐ」上級戦略が存在する。

### 2.3 お邪魔オブジェクト(`obstacles`)

蝶ともアイテムとも異なる第 3 の種別軸。設定 `obstacles: string[]` で種類を
指定し、ステージ時間を等分したタイミング(+3 秒、花とずれる)で各 1 体が出現する。
ボーナスステージには出現しない。

| 種類              | 動き                         | 描画中の線に触れると                                                        | 捕獲ループに含めると                                                                             |
| ----------------- | ---------------------------- | --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| いもむし (catapy) | 最遅 (0.2)。ほぼ水平に這う   | 無害                                                                        | **ループ全体が無効**(捕獲・色替え・花取得なし。減点もなし)。体が少しでも入れば「入っている」判定 |
| ハチ (bee)        | 最速 (4.8)。長い直線ジグザグ | **線の寿命が 1/3(0.5 秒)に 6 秒間短縮**(線が赤くなる)。発動後は画面外へ退場 | **影響なし**(蝶の捕獲は通常どおり成立する)                                                       |
| クモ (spider)     | 遅い (0.35)。ゆっくり徘徊    | **6 秒間、全蝶が鉛筆カーソルから逃げる**。発動後は退場                      | **影響なし**(同上)                                                                               |

共通仕様:

-   catapy は退場せず**常駐**する。bee / spider は効果を一度発動すると退場するため、
    1 ステージの妨害は「単発イベント×種類数+catapy の常駐」の構造になる。
-   **蝶を含まないループで単体を囲むとカウントが進み、3 回で消える**(全種共通)。
    「3 回」という数字は隠し仕様として How To Play に記載しない方針だが、
    囲むたびにひるむ演出は実装済みであり、§4.3-5 の進捗フィードバックを
    加えることで「効いている」ことだけは伝わるようにする。

難易度への効き方と対処はそれぞれ異なる:

-   **catapy = 空間の制約**。ループを引ける場所を狭める。常駐するため
    ステージ全体の難易度を底上げする。基本対処は「離れた場所で囲む」、
    発展対処は「3 回囲んで排除」。
-   **bee = 時間の制約**。速いが、線に触れた瞬間に効果を使い切って退場する
    **1 回きりの時限爆弾**。基本対処は「進路から離れた場所・タイミングで描く」。
    発展対処は**「毒見(おとり線)」— 蝶を狙っていない安全な瞬間にわざと短い
    捨て線を触れさせ、被害ゼロで退場させる**。これが決まると以降はクリーンな
    盤面になる。
-   **spider = 操作の制約**。放置して線に触れられると 6 秒失うが、遅いので
    「線を近づけない」だけでも凌げる。発展対処は空ループ 3 回での先制排除。

bee と spider は「凌ぐ」だけでもクリアできる need に設定し(§5)、
排除・毒見は知っている人が楽になる**上級の近道**に位置づける。
隠し仕様を攻略の前提にはしない。

### 2.4 お助けフラワー

`helpObjectNum` 個が、ステージ時間を等分したタイミングで**ランダムな種類**で降ってくる。

| 種類      | 効果                           | 持続  |
| --------- | ------------------------------ | ----- |
| freeze    | 蝶が停止する                   | 3 秒  |
| time_plus | 経過時間を 5 秒巻き戻す        | 即時  |
| gather    | 蝶が**色ごとの集合点**に集まる | 10 秒 |
| long      | 線の寿命 +0.5 秒・線色変化     | 5 秒  |

gather は「同色まとめ捕り」を強烈に補助するが、種類はランダムであるため
**特定の花が来ることを前提にした難易度設計はしない**(花 3 個で gather が
1 個も来ない確率は約 42% ある)。花は「来たら加速する装置」と位置づけ、
クリアラインは花なしでも届く水準に置く(§3)。

### 2.5 ステージ進行と失敗コスト

-   1 ステージ 60 秒。`needCount` 匹捕獲でクリア、時間切れで未達なら失敗。
-   **失敗するとランが終了し、タイトルに戻る**(ステージ単位のリトライは無い)。
    記録されるのはラン合計スコアの自己ベストのみ(`ScoreStorage`)。
-   つまり本作は**ラン型(アーケード型)**であり、各ステージのクリア率は
    直列に掛け算される。「Lv20 に到達できる確率 = 全ステージクリア率の積」
    である点を、クリア率目標(§7)は織り込まなければならない。
-   レベルが 5 の倍数のステージにスペシャル蝶が出る(`level % 5 === 0`)。

## 3. 難易度パラメータの効き方

設計で操作できる 9 つのパラメータと、難易度への寄与を整理する。

| パラメータ               | 上げると    | 主な効き方                                                                             |
| ------------------------ | ----------- | -------------------------------------------------------------------------------------- |
| `needCount`              | 難化        | 要求捕獲ペースが直接上がる(下表)                                                       |
| `stageButterflyCount`    | **両面**    | 多いと同色候補が増えまとめ捕りしやすいが、ループへの「混入事故」も増える               |
| `butterflyColorNum`      | 難化        | 同色ペアの発生率がほぼ 1/色数 で低下。3 色以上ならコンプ捕り(×20)が解禁                |
| `butterflySize` small    | 難化        | 速くて判定が小さい。large は遅いが体が大きく、大きなループを強いる                     |
| `isButterflyColorChange` | 難化+戦略化 | 色の視認が難しくなる一方、単騎ループの色替えが戦略資源になる                           |
| `multipleButterflyRate`  | 報酬増      | 難易度はほぼ変えずスコア上限を引き上げる(「狙う楽しさ」の変数)                         |
| `maxMultipleRate`        | 報酬増      | x3 は同色群に混ぜたときの爆発力が大きい                                                |
| `helpObjectNum`          | 緩和        | 救済と得点チャンス。難しいステージほど多めに配る                                       |
| `obstacles`              | 難化        | catapy=空間、bee=時間、spider=操作の制約(§2.3)。1 種につき有効ループ数を約 1〜2 回削る |

### 捕獲ペースの目安(クリア可能性の見積もり)

ループ 1 回の所要は描画 1.5 秒+位置取りで 3〜5 秒。ただし実プレイには
空振り・失敗ループ・蝶待ち・花の回収が挟まるため、60 秒の**有効ループ数**は
次の 2 水準で見積もる。

-   熟練プレイヤー: **12〜15 回**
-   初見〜カジュアル: **8〜10 回**

さらに**お邪魔オブジェクトが 1 種いるごとに 1〜2 回差し引く**
(bee/spider の 6 秒妨害、catapy の回避コスト)。

クリアラインの基準は**二段構え**にする。フェーズ 1〜3(Lv1-14)は
**カジュアル水準で届くこと**、フェーズ 4(Lv15-20)は**熟練水準で届くこと**を
基準にする(ラン型ゆえ、フェーズ 4 に到達するのは自己選抜された熟練者だけで
あるため)。各基準と needCount の差分は超過ボーナス・スコアメイクの余地として残す。

| needCount | 必要平均(カジュアル水準基準) | 体感                                                                                       |
| --------- | ---------------------------- | ------------------------------------------------------------------------------------------ |
| 〜10      | 1.0                          | ペア捕りだけで届く。初心者帯                                                               |
| 12〜14    | 1.2〜1.4                     | ペア中心+時々 3 匹。中級帯                                                                 |
| 16〜18    | 1.6〜2.0                     | 3 匹前後のまとめ捕りが常時必要                                                             |
| 20〜24    | 2.2〜3.0                     | カジュアル水準では壁。熟練水準(分母 10〜12 回)なら平均 2.0〜2.4 で届く — フェーズ 4 の基準 |
| 28 超     | 3.0+                         | 熟練水準でも困難(エクストラ帯の到達目標)                                                   |

この表が本設計の背骨である。ただし有効ループ数は仮説値なので、
**§7 の検証で最優先に実測し、ズレていれば needCount を一括補正する**。
`needCount` を上げるステージ、お邪魔を足すステージでは、まとめ捕りを
成立させる手段(蝶の数・少色数・花の個数)を必ずセットで与える。

## 4. 難易度カーブの方針

### 4.1 4 フェーズ+エクストラの構成(本編 = Lv1〜20)

**レベル 20 を本編の実質的な最終到達点(体感 MAX)とする。**
レベル 21 以降は Lv20 の構成に 3 種お邪魔を足し、needCount だけが伸びる
エクストラ帯(§5.3)。

```
難易度
  ▲                                       ★20
  │                          ★15 ↗17↗18↗19
  │                ★10 ↗11↗ ＼13 14
  │       ↗ 6 ↗ 7 ↗ 9 ＼   12
  │ 1→2↗3↗4 ★5     8
  └──────────────────────────────────────▶ レベル
    学習     応用      お邪魔との遭遇   マスタリー
   (1-3)    (4-8)       (9-14)        (15-20)
```

-   **フェーズ 1: 学習 (Lv1-3)** — ループの引き方、同色捕り、コンプ捕りを 1 つずつ教える。
-   **フェーズ 2: 応用 (Lv4-8)** — 色替え・スペシャル蝶・倍率蝶・多色・large を順に導入する。
-   **フェーズ 3: お邪魔との遭遇 (Lv9-14)** — いもむし→ハチ→クモを 1 種ずつ導入し、
    既習の蝶種別と組み合わせる。
-   **フェーズ 4: マスタリー (Lv15-20)** — 5 色・全お邪魔・全種別の総合試験。
    20 をグランドフィナーレとする。

### 4.2 緩急の原則

1. **1 ステージ 1 テーマ**: 新要素(蝶種別もお邪魔も)は 1 つずつ。既知の要素との
   組み合わせは翌ステージ以降。新メカニクス導入ステージは needCount を
   前ステージから**据え置く**(数字に追われると新しい遊びを試さないため)。
2. **山の直後に谷**: 難しいステージの次は「派手に捕れる」ステージを置き、
   爽快感で回復させる(8, 12, 17 は蝶数が多い)。
3. **5 の倍数=祝祭**: スペシャル蝶ステージ(5, 10, 15, 20)は難易度を半歩下げ、
   「スペシャル蝶を探して確保する」ことに意識を割けるようにする。
   祝祭ステージでは新しいお邪魔を導入しない。
4. **needCount は非減少**: プレイヤーの上達実感の軸として一貫させる。
   ただし単調増加は強制せず、テーマが難しいステージでは据え置きを許す。
5. **ラン型を忘れない**: 失敗=ラン終了なので、中盤以降の 1 ステージの
   クリア率を欲張って下げると Lv20 到達者がいなくなる(§7 の累積計算参照)。
6. **お邪魔は「常駐 1 + 単発 1」まで**: 本編は 2 種同時を上限とし、
   3 種同時は**エクストラ帯(Lv21 以降)だけの特権**にする。妨害の飽和は
   「難しい」ではなく「うっとうしい」に転じるため。また原則 3 と同様、
   一度も練習していない組み合わせをフィナーレに置かない。

### 4.3 ゲームが説明しない「5 つの暗黙ルール」(実装依存事項)

プレイテストレビューで、config 調整だけでは解決できない伝達課題が
特定された。**本設計のクリア率目標はこれらの手当てを前提とする。**
未対応のままだと、カジュアル層は「理由の分からない失敗」を重ねて離脱する。

| #   | 暗黙ルール                                              | 必要な手当て(案)                                                                                                                                                                                               |
| --- | ------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 異色 2 匹は失敗(−20)                                    | 失敗時に理由を表示する(例:「ちがう色 2 匹はダメ!」)。現状の "Bad Loop!" だけでは理由が伝わらない                                                                                                               |
| 2   | 1 匹囲み=色替え                                         | 色替え発動時のエフェクト強化と、Lv4 開始時のヒント表示                                                                                                                                                         |
| 3   | スペシャル蝶を捕る前に needCount 到達するとボーナス消滅 | ボーナスを逃したリザルトで「光る蝶を捕まえてからクリアするとボーナスステージ!」と明示する                                                                                                                      |
| 4   | いもむしを含むループは無効                              | "Invalid loop!" 表示に加え、**原因の catapy を一瞬ハイライト**する。どの個体のせいで無効になったかが分からないと理不尽に感じる                                                                                 |
| 5   | お邪魔は空ループで囲むと消せる(3 回)                    | 囲むたびのひるみ演出は実装済み。加えて**進捗が伝わるフィードバック**(ひるみの段階的強調やカウント表示)を入れる。「3 回」の明文化は不要だが、「効いている」ことが伝わらないと試行が強化されず、誰も発見できない |

なお「蝶は線に触れると反転する」は全サイズ共通の視覚的に自然な挙動であり
(初版で「large は遠くから感知する」としていたのは誤り。§8.2)、
暗黙ルールからは除外した。#5 は「3 回で消える」という正解の明文化ではなく、
発見の喜びを残したまま**発見可能にする**ための手当てである。

これらは GitHub Issue として個別に切り出すことを推奨する。

### 4.4 ラン構造への手当て(要コード変更・Issue 候補)

第 2 回レビューで、ステージ単体ではなく**ラン構造の経済**が最大の離脱要因に
なり得ると指摘された。失敗=ラン終了の 20 ステージ構成は、フェーズ 3 で落ちる
多数派に「覚え切った Lv1〜8 の再走」を毎回課し、1 ラン最大 20 分超は
カジュアル層の 1 セッション(5〜10 分)に収まらない。以下を config 外の
実装課題として推奨する。

1. **練習スタート(推奨)**: 到達済みのフェーズ境界(Lv9 / Lv15)から開始できる
   モードを設ける。練習スタートのランはハイスコア記録の対象外とすれば、
   ラン型スコアアタックの公平性は保てる。
2. **序盤の高速通過を仕様として保証する**: needCount 到達で即ステージ終了する
   現仕様は再走の味方である(熟練者は Lv1-3 を各 20〜30 秒で抜けられる)。
   §7.1 の実測に「熟練者の Lv1-8 再走所要時間」を含め、リザルト画面込みで
   5 分を大きく超えるようなら序盤の need をさらに下げる。
3. **完走率を設計目標として持つ**: 単発ランの完走確率(§7.2 の累積 0.8%)とは
   別に、「継続プレイヤーの 10% がリトライを重ねていつかは Lv20 に到達する」を
   目標に置く。下回る場合の是正順序は §7.2 に定める。

## 5. ステージ別設計(レベル 1〜20)

### 5.1 一覧表

★ = スペシャル蝶出現。「倍率」は `multipleButterflyRate` / `maxMultipleRate`。
お邪魔列: 🐛 = catapy / 🐝 = bee / 🕷 = spider。

| Lv  | テーマ              | size   | 色数 | 色替え | need | 蝶数 | 倍率      | 花  | お邪魔 | 難度  |
| --- | ------------------- | ------ | ---- | ------ | ---- | ---- | --------- | --- | ------ | ----- |
| 1   | はじめてのループ    | large  | 2    | —      | 6    | 8    | —         | 0   | —      | ☆     |
| 2   | ちいさな群れ        | small  | 2    | —      | 8    | 16   | —         | 1   | —      | ☆     |

### 5.2 各ステージの特徴と想定戦略

#### フェーズ 1: 学習 (Lv1-3)

**Lv1「はじめてのループ」** — 遅い large 蝶が 8 匹、2 色のみ。花なし。
要求 6 匹はペア捕り 3 回で足りる。近くの同色 2 匹をゆっくり囲むだけで勝てる。
学習目標はループを閉じる操作と「同色なら捕れる」ルール、そして
「蝶に線が触れると逃げる」こと(largeは遅いので安全に観察できる)。
失敗しても −20 点が軽い傷で済む蝶数にしてある。

**Lv2「ちいさな群れ」** — 最速の small が 16 匹。2 色なので画面は同色だらけ。
速くて 1 匹ずつは追えないが、16 匹もいればどこを囲っても同色が 2〜3 匹入る。
学習目標は「数が多ければまとめて捕れる」ことと、お助け花(初登場)を取ること。
Lv1 と逆の性質を体験させ、サイズごとに手触りが違うことを刷り込む。

**Lv3「三色の庭」** — 3 色になり同色ペアが見つけにくくなる。
**3 色 1 匹ずつの「コンプ捕り」(×20 点)** が解禁され、「異色 2 匹だけは
絶対に囲まない」判断を学ぶ。§4.3-1 の失敗理由表示に強く依存するステージ。

#### フェーズ 2: 応用 (Lv4-8)

**Lv4「いろがわりの午後」** — 色替え蝶が初登場。邪魔な位置の 1 匹を
わざと単騎ループして**色を変え、同色群を製造する**ことを学ぶ。
need を据え置き(10)、数字に追われず色替えで遊ぶ余白を作る。
§4.3-2 のヒント表示を前提とする。

**Lv5「★ひかりの予感」** — 開始 10 秒で金色に光る蝶が出現。倍率蝶(x2)も
低頻度で混ざり始める。スペシャル蝶は**同色の仲間と一緒に囲んで確保**
(単騎では色替え扱いで捕れない)。**初見は高確率で先にクリアしてしまい
ボーナスを逃す**が、それは仕様上避けられない — 逃した直後のリザルトで
種明かしして(§4.3-3)、Lv10 でのリベンジ動機に変換する。

**Lv6「おおものラッシュ」** — large × 色替え。蝶数 10 とやや少ない。
large は体が大きいので、収めるには**大きなループ**が要り、描いている間に
線に触れて逃げられやすい。先回りして大きな円を先に描き始め、蝶が線に
触れる前に閉じる。色替えで同色 3 匹を作ってから一網打尽が理想形。
フェーズ 2 の山その 1。

**Lv7「四色の乱舞」** — 4 色。同色ペアの自然発生が急落し、
**バラバラ 3〜4 匹のコンプ捕り**が主力になる。「同色狙い」から
「コンプ狙い」への戦略スイッチを確立させる。フェーズ 2 の山その 2
なので need は 14 に据え置き。

**Lv8「ちいさな嵐」** — 高速 small 16 匹+色替え。x3 倍率蝶が初登場。
追いかけず、群れが重なる場所に**先に線を引いて待ち構える**(Lv2 の上位互換)。
x3 蝶入りの同色 4 匹 = 120 点のスコアメイクを覚える。お邪魔導入前の
爽快な谷。

#### フェーズ 3: お邪魔との遭遇 (Lv9-14)

**Lv9「のんびりやの闖入者」** — **いもむし(catapy)初登場**。ゆっくり
水平に這うだけだが、**一緒に囲むとループが無効**になる。色数 3・色替えなしと
蝶側の条件を Lv3 相当まで戻し、「囲んではいけないものがいる」という
1 点だけを学ばせる。想定戦略は「いもむしから離れた場所で囲む」。
気づいたプレイヤーは単体 3 回囲みで排除できる(隠し仕様の発見)。
need は 16 に据え置き。§4.3-4 の原因ハイライトを前提とする。

**Lv10「★はちみつ収穫祭」** — 祝祭。3 色・蝶数 16・倍率 20%/x3・花 4 と
大盤振る舞いで、お邪魔は慣れたいもむし 1 体のみ。Lv5 でボーナスを逃した
プレイヤーの**リベンジ戦**として「スペシャル蝶を先に確保 → クリア」の手順を
試せる。倍率蝶入り同色 5 匹(x3 = 150 点)の収穫コンボや、クリア直前で
止めて最後に大量捕獲 → 超過ボーナス 100 点/匹の練習場。

**Lv11「はたらきもののハチ」** — **ハチ(bee)初登場**。高速ジグザグで
飛び回り、描画中の線に触れると**線の寿命が 6 秒間 1/3 になる**(線が赤くなる)。
学びは「描くタイミングの管理」— ハチの進路を見て、離れた隙に引く。
被弾したら 6 秒は無理をせず蝶の配置を読む時間に充てる。効果は 1 回きりで
発動後にハチは退場するため、**毒見(おとり線。§2.3)で先に無力化する**
発展戦略に気づけると、以降のハチ登場ステージの景色が一変する。
蝶側は Lv10 と同じ 3 色・色替えで認知負荷を抑え、need も 18 に据え置き。

**Lv12「二匹の来客」** — いもむし+ハチの 2 種同時(どちらも既習)。
その代わり蝶は small 18 匹・2 色の物量天国で、どこを囲っても同色群がいる。
「妨害をさばきながら爽快にまとめ捕り」のリズムを作る谷ステージ。
倍率 20%/x3 で 1 ループ 5 匹 × x3 = 150 点級のチャンスも多い。

**Lv13「あみはりのクモ」** — **クモ(spider)初登場**。遅い徘徊だが、線に
触れられると**6 秒間、全蝶が鉛筆カーソルから逃げる**。学びは 2 段階:
(1) クモに線を触れさせない、(2) 触れられたら 6 秒間は追わない。
need 20 は**この 2 つだけで届く水準**にする — 排除(§2.3)を知らない
プレイヤーが別のゲームを遊ぶことにならないように。空ループでの先制排除に
気づいた人はさらに楽になり、上級者は効果中の「蝶が鉛筆から逃げる」を
逆手に取って**牧羊犬のように蝶を隅へ追い込んでから囲む**荒業もある。
4 色でコンプ捕り主体。

**Lv14「いたずらな羽色」** — random サイズ × 4 色 × 色替え × ハチ+クモ。
フェーズ 3 の卒業試験で、画面の情報量が最大級。主力は 3 色コンプ捕り、
色替えの単騎ループで素材を「調達」できると余裕が生まれる。
お邪魔 2 種は単発型(bee/spider)なので、凌げば終盤はクリーンな盤面になる。

#### フェーズ 4: マスタリー (Lv15-20)

**Lv15「★ひかりの楽園」** — 祝祭。3 色に戻し、蝶数 16・倍率 20%/x3・花 4。
お邪魔はいもむし 1 体のみ。フェーズ 3 を越えたプレイヤーへのご褒美として
スコアが伸びる設計で、スペシャル蝶からのボーナスステージで貯金を作る。

**Lv16「五色の森」** — **全 5 色**が出揃う。同色ペアは滅多に並ばず、
**コンプ捕り(3〜4 色)がほぼ唯一の安定打**。gather 花を引けたら色の島が
でき、同色まとめ捕りがボーナスタイムになる。コンプ捕りは散らばった異色を
まとめて囲むため線が長くなり、ハチの被弾リスクが構造的に高い —
**毒見(§2.3)で先に無力化してから色に集中する**のが安定択。
need 22 据え置きで、色の難しさだけで勝負する。

**Lv17「群れの王国」** — small 20 匹・2 色のみ。画面が蝶で埋まる最大の
爽快枠。**1 ループ 5 匹以上**を狙い続ける物量戦で、2 色なのでコンプ捕りは
存在せず、混色事故だけが敵。クモが 1 体這っているが、効果は単発 6 秒なので
**被弾しても物量で押し切れる need にしてある**(排除は前提にしない)。
画面が蝶で埋まっていて空ループの隙間が少ないため、排除を狙うなら蝶の薄い
画面端で — これ自体が Lv13 の学びの応用問題になっている。

**Lv18「五色乱舞」** — 5 色 × 色替え × いもむし+ハチ。本作の
「色システム」の集大成にして最難関その 1。3〜4 色コンプ捕りを主軸に、
単騎色替えで素材を調達する。×20 の単価が高いのでコンプが回れば得点効率は
むしろ良い。need 24 は色替えなしでも理論上届くが、失敗の許容回数がほぼない。

**Lv19「おおものの森」** — large 13 匹 × 4 色 × 色替え × ハチ+クモ。
大きなループを先読みで引く技術の集大成、最難関その 2。大きな円を描く=
線を長時間さらすことなので、ハチの被弾リスクが構造的に高い。
**出現したら毒見(§2.3)で先に処理してから大きな円に取りかかる**のが推奨手順。
freeze 花の価値が最大(large の遅さが完全に無力化される)。
「色の 18」と「操作の 19」で最終試験を分け、落ちたとき自分の弱点が分かる。

**Lv20「★グランドフィナーレ」** — 集大成の祝祭。random・4 色・色替え・
倍率 20%/x3・花 5・スペシャル蝶。お邪魔は**既習ペアのいもむし+ハチ**
(Lv18 と同じ組み合わせ)に留める — 祝祭原則(§4.2-3)どおり、一度も
練習していない構成をフィナーレに置かない。3 種同時の「地獄」は
エクストラ帯(§5.3)の看板に取っておく。
難度は 18/19 より明確に下げる(need 据え置き 24、蝶数 16、花 5)。
「試験の後の卒業式」として、ここまで来たプレイヤーの大半が気持ちよく
終われるクリア率(§7 で 70%)を目標にする。スペシャル蝶を確保して
ボーナスステージで有終の美を飾るのが理想の結末。

### 5.3 レベル 21 以降(エクストラ帯: 限界挑戦)

**Lv20 を体感上の最終レベルとし、21 以降は「Lv20 の構成+3 種お邪魔同時+
needCount +2/レベル」の限界挑戦帯とする。**「エンドレス」とは呼ばず
**エクストラ**と呼び、無限に新しいコンテンツが続くかのような期待を持たせない。

-   現行実装の 4 パターン巡回生成(群れ/カラフル/大物/混沌)は**廃止**し、
    最終エントリ(Lv20)を引き継いで needCount のみ加算する方式に置き換える
    (`generateOverflowConfig` の簡素化。小さなコード変更で済む)。
    お邪魔だけは Lv21 から**3 種同時(🐛🐝🕷)**に増やし、「本編を卒業した者
    だけが見る景色」という質的な一段を残す。
-   根拠: (1) 全員が同じ Lv21・Lv22 を走るため、ラン合計スコアの比較が公平に
    なる(4 パターン巡回は引き次第でスコアがブレていた)。(2) 自動生成の
    4 パターンは本編 1〜20 に吸収済みで、役割を終える。(3) 実装・テスト・
    バランス検証の対象が 1 構成に絞られる。
-   §3 の見積もりから need 26(Lv21)〜30(Lv23)で急速に達成困難になり、
    **Lv22〜23 が事実上の到達限界**となる。これは欠陥ではなく「限界が来ることが
    確定している限界挑戦」として割り切る。やり込みの軸は到達レベル単独ではなく
    **ラン合計スコアとの複合**に置く(同じ Lv22 で終わっても道中の稼ぎで差がつく)。
-   Lv25 のスペシャル蝶+ボーナスステージまで届いたら、それは「花火の向こう側」
    — 実質的な称号である。

## 7. 検証計画

数値は仮説であり、プレイテストで以下を計測して調整する。
検証は `stage-config-debug.json` に §5.1 の値を写して `npm run debug` で行う
(お邪魔の検証は `feature/obstacle-objects` ブランチ上で行う)。

### 7.1 最優先: 背骨の実測

**有効ループ数(60 秒あたりの意味のあるループ回数)を初見層/熟練層で実測する。**
あわせて**お邪魔 1 種あたりの有効ループ削減数**(仮説: 1〜2 回)と、
**熟練者の Lv1-8 再走所要時間**(目標: リザルト込み 5 分以内。§4.4-2)も計測する。
§3 の仮説が楽観側にズレていた場合、全ステージの needCount を一括で −2
補正するフォールバックを用意しておく。

### 7.2 ステージ別クリア率

ラン型(失敗=ラン終了)なので、クリア率は直列に掛かる。目標:

| 帯      | ステージ別クリア率目標 | 累積到達率(目安) |
| ------- | ---------------------- | ---------------- |
| Lv1-3   | 95%+                   | Lv4 到達 86%     |
| Lv4-8   | 85%                    | Lv9 到達 38%     |
| Lv9-14  | 75%                    | Lv15 到達 7%     |
| Lv15-19 | 70%                    | Lv20 到達 1.2%   |
| Lv20    | 70%                    | 完走 0.8%        |

累積 0.8% は「1 ランで一気に完走する確率」であり、リトライを重ねて上達する
前提の数字である(中盤以降の到達者は上達による自己選抜がかかるため、
実測のステージ別クリア率は表より高く出るはず)。単発ランの確率とは別に、
**「継続プレイヤーの 10% が、リトライを重ねていつかは Lv20 に到達する」を
完走率の設計目標**として持つ(§4.4-3)。実測が目安を大きく下回る場合の
是正は、needCount 引き下げ → 花の増量 → 練習スタート導入(§4.4-1)の
順で行う。

### 7.3 その他の計測指標

1. **バッドループ率**(失敗ループ/全ループ): 20% 超のステージは色数か蝶数を見直す。
   §4.3-1(失敗理由表示)の実装前後で比較できるとなお良い。
2. **gather 有無別のクリア率**: 差が 15pt を超えるステージは「gather 引けたか
   どうかの運ゲー」になっている兆候。needCount を下げるか蝶数を増やす。
3. **お邪魔別の影響計測**: catapy による無効ループ率(Lv9 で高すぎるなら
   §4.3-4 の伝達不足)、bee の被弾率と毒見(意図的発動)率、
   spider・catapy の排除率(§4.3-5 のフィードバックで隠し仕様が
   発見されているか)。排除率・毒見率が数%に留まるなら、フィードバックの
   強化かヒント表示を検討する。
4. **1 ループ平均捕獲数**: §3 の必要平均を実測が下回るステージは成立手段を追加。
5. **お助け花の取得率**: 50% を大きく下回る場合、花の落下位置・速度の調整を検討
   (本書のスコープ外だがボトルネックになり得る)。
6. **Lv5 でのボーナス取り逃し率**: 高くて構わない(初回は逃す前提)が、
   Lv10 での回収率が低い場合は §4.3-3 の伝達が機能していない。

### 8.2 実装再確認による訂正(2026-07-09)

初版の「large は線を遠くから感知して反転する」という記述は誤りだった。
実測では線感知半径は全サイズで体の半径の約 55%(large 14px/体半径 26px、
small 9px/15px)であり、**「線が体に触れたら反転する」全サイズ共通の挙動**である。
large の囲いにくさは感知ではなく体の大きさ(必要ループ径と捕獲判定円の大きさ)に
由来する。§2.1・Lv6/Lv19 の設計意図・§4.3 を全て訂正済み。
第 1 回レビュー指摘「線感知の不可視性」も、実態が視覚的に自然な挙動のため
大部分が解消と判断した(catapy の無効化理由表示 §4.3-4 は引き続き必要)。
