# LoopGame ステージ難易度設計書

Stage Difficulty Design Document

## 1. 目的とスコープ

本書は LoopGame のステージ難易度カーブを再設計するための提案書である。
現行実装が持つ「蝶の種別」— サイズ 3 種・スペシャル蝶・色替え蝶・倍率蝶 — を
**最大限に活かし**、各ステージが固有の「学び」と「戦略」を持つように
レベル 1〜15 の構成と、それ以降(エンドレス帯)の方針を定義する。

主対象は `src/scripts/utils/stage-config.json` の設定値であり、ステージ構成
そのものは**コード変更なし**で実現できる。ただしプレイテストレビューの結果、
ルール伝達に関する演出面の依存事項が明らかになったため、§4.3 に別途列挙する
(これらは config だけでは解決できず、実装課題として切り出すべきもの)。

## 2. 設計の前提: 現行メカニクスの整理

### 2.1 蝶の種別

| 種別    | 速度 (x/y) | 線感知半径 | 特徴                                                                                                                               |
| ------- | ---------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| small   | 0.6 / 0.6  | 9          | 最速。動きが読みにくく、囲うのに先読みが要る                                                                                       |
| medium  | 0.5 / 0.4  | 11         | 標準。バランス型                                                                                                                   |
| large   | 0.4 / 0.3  | 14         | 最遅だが**線を遠くから感知して反転する**ため、雑な小さいループでは逃げられる                                                       |
| special | 0.6 / 0.5  | sprite高/3 | レベル 5 の倍数で開始 10 秒後に 1 匹だけ出現。金色に発光。捕獲判定が緩い (hitRate 0.4)。**捕獲してクリアするとボーナスステージへ** |

サイズとは独立に、次の 2 つの「種別修飾」がある。

-   **色替え蝶** (`isButterflyColorChange`): メイン色と別のサブ色を持つ。
    **1 匹だけをループで囲むと色がサブ色に切り替わる**(捕獲されない)。
-   **倍率蝶** (`multipleButterflyRate` / `maxMultipleRate`): 葉っぱに「x2」「x3」が
    表示され、含めて捕獲すると**そのループの得点全体が乗算される**。
    複数含めれば乗算が重なる(x2 と x3 なら ×6)。

### 2.2 捕獲ルール

1 回のループで囲んだ蝶の内訳により結果が分岐する。

| 囲んだ内容                          | 結果                                                       |
| ----------------------------------- | ---------------------------------------------------------- |
| 0 匹                                | 何も起きない(花のみ取得可)                                 |
| 1 匹                                | 捕獲ではなく**色替え**(色替え蝶でない場合は見た目変化なし) |
| 2 匹以上・全員同色                  | 捕獲成功。**匹数 × 10 点**                                 |
| 3 匹以上・**全員バラバラの色**      | 捕獲成功。**匹数 × 20 点**                                 |
| 上記以外(混色で重複あり、異色 2 匹) | **失敗。−20 点**                                           |

-   線は引いてから **1.5 秒**で消える(ロングループ中は 2.0 秒)。
-   蝶は引いた線に触れると反転して逃げる(線=柵として使える)。
-   捕獲した分だけ新しい蝶が即座に補充される(スペシャル蝶は補充されない)。
-   `needCount` に到達した瞬間にステージ終了。**超過分は 1 匹 100 点のボーナス**
    になるため、「最後のループで大量捕獲して超過を稼ぐ」上級戦略が存在する。

### 2.3 お助けフラワー

`helpObjectNum` 個が、ステージ時間を等分したタイミングで**ランダムな種類**で降ってくる。

| 種類      | 効果                           | 持続  |
| --------- | ------------------------------ | ----- |
| freeze    | 蝶が停止する                   | 3 秒  |
| time_plus | 経過時間を 5 秒巻き戻す        | 即時  |
| gather    | 蝶が**色ごとの集合点**に集まる | 10 秒 |
| long      | 線の寿命 +0.5 秒・線色変化     | 5 秒  |

gather は「同色まとめ捕り」を強烈に補助するが、種類はランダムであるため
**特定の花が来ることを前提にした難易度設計はしない**(花 3 個で gather が
1 個も来ない確率は約 42% ある)。花は「来たら加速する装置」と位置づけ、
クリアラインは花なしでも届く水準に置く(§3)。

### 2.4 ステージ進行と失敗コスト

-   1 ステージ 60 秒。`needCount` 匹捕獲でクリア、時間切れで未達なら失敗。
-   **失敗するとランが終了し、タイトルに戻る**(ステージ単位のリトライは無い)。
    記録されるのはラン合計スコアの自己ベストのみ(`ScoreStorage`)。
-   つまり本作は**ラン型(アーケード型)**であり、各ステージのクリア率は
    直列に掛け算される。「Lv15 に到達できる確率 = 全ステージクリア率の積」
    である点を、クリア率目標(§7)は織り込まなければならない。
-   レベルが 5 の倍数のステージにスペシャル蝶が出る(`level % 5 === 0`)。
-   設定 JSON を超えたレベルは 4 パターン(群れ/カラフル/大物/混沌)を巡回する
    自動生成となり、`needCount` はレベルごとに +2 され続ける(エンドレス帯)。

## 3. 難易度パラメータの効き方

設計で操作できる 8 つのパラメータと、難易度への寄与を整理する。

| パラメータ               | 上げると    | 主な効き方                                                                   |
| ------------------------ | ----------- | ---------------------------------------------------------------------------- |
| `needCount`              | 難化        | 要求捕獲ペースが直接上がる(下表)                                             |
| `stageButterflyCount`    | **両面**    | 多いと同色候補が増えまとめ捕りしやすいが、ループへの「混入事故」も増える     |
| `butterflyColorNum`      | 難化        | 同色ペアの発生率がほぼ 1/色数 で低下。3 色以上ならコンプ捕り(×20)が解禁      |
| `butterflySize` small    | 難化        | 速くて判定が小さい。large は遅い代わりに線感知が早く、大きなループを強要する |
| `isButterflyColorChange` | 難化+戦略化 | 色の視認が難しくなる一方、単騎ループの色替えが戦略資源になる                 |
| `multipleButterflyRate`  | 報酬増      | 難易度はほぼ変えずスコア上限を引き上げる(「狙う楽しさ」の変数)               |
| `maxMultipleRate`        | 報酬増      | x3 は同色群に混ぜたときの爆発力が大きい                                      |
| `helpObjectNum`          | 緩和        | 救済と得点チャンス。難しいステージほど多めに配る                             |

### 捕獲ペースの目安(クリア可能性の見積もり)

ループ 1 回の所要は描画 1.5 秒+位置取りで 3〜5 秒。ただし実プレイには
空振り・失敗ループ・蝶待ち・花の回収が挟まるため、60 秒の**有効ループ数**は
次の 2 水準で見積もる。

-   熟練プレイヤー: **12〜15 回**
-   初見〜カジュアル: **8〜10 回**

クリアラインは**カジュアル水準(10 回)で届くこと**を基準にし、熟練水準との
差分は超過ボーナス・スコアメイクの余地として残す。

| needCount | 必要平均(10 ループ基準) | 体感                                               |
| --------- | ----------------------- | -------------------------------------------------- |
| 〜10      | 1.0                     | ペア捕りだけで届く。初心者帯                       |
| 12〜14    | 1.2〜1.4                | ペア中心+時々 3 匹。中級帯                         |
| 16〜18    | 1.6〜1.8                | 3 匹前後のまとめ捕りが常時必要                     |
| 20〜24    | 2.0〜2.4                | まとめ捕り必須+花をフル活用。上級者の壁            |
| 28 超     | 2.8+                    | 理論上可能だが常人には困難(エンドレス帯の到達目標) |

この表が本設計の背骨である。ただし有効ループ数は仮説値なので、
**§7 の検証で最優先に実測し、ズレていれば needCount を一括補正する**。
`needCount` を上げるステージでは、まとめ捕りを成立させる手段
(蝶の数・少色数・花の個数)を必ずセットで与える。

## 4. 難易度カーブの方針

### 4.1 4 フェーズ構成

```
難易度
  ▲                                        ★15
  │                              ★10  ↗14
  │                    ★5   ↗ 8 ↗  ＼11↗
  │          ↗ 3 ↗ 4 ↗  ＼6 ↗ 9    12
  │  1 → 2 ↗            7
  └──────────────────────────────────────▶ レベル
     学習      応用       種別の掛け算   マスタリー
    (1-3)     (4-7)        (8-11)       (12-15)
```

-   **フェーズ 1: 学習 (Lv1-3)** — ループの引き方、同色捕り、コンプ捕りを 1 つずつ教える。
-   **フェーズ 2: 応用 (Lv4-7)** — 色替え・スペシャル蝶・倍率蝶・多色を順に導入する。
-   **フェーズ 3: 種別の掛け算 (Lv8-11)** — 導入済みの種別を 2〜3 個ずつ組み合わせる。
-   **フェーズ 4: マスタリー (Lv12-15)** — 全種別の総合試験。15 をフィナーレとする。

### 4.2 緩急の原則

1. **1 ステージ 1 テーマ**: 新要素は 1 つずつ。既知の要素との組み合わせは翌ステージ以降。
   新メカニクス導入ステージは needCount を前ステージから**据え置く**
   (数字に追われると新しい遊びを試さないため)。
2. **山の直後に谷**: 難しいステージ(7, 11, 14)の次は「派手に捕れる」ステージを置き、
   爽快感で回復させる(8, 12, 15 は蝶数が多い)。
3. **5 の倍数=祝祭**: スペシャル蝶ステージ(5, 10, 15)は難易度を半歩下げ、
   「スペシャル蝶を探して確保する」ことに意識を割けるようにする。
   ボーナスステージ突入が最大の報酬なので、クリア自体を難しくしすぎない。
4. **needCount は非減少**: プレイヤーの上達実感の軸として一貫させる。
   ただし単調増加は強制せず、テーマが難しいステージでは据え置きを許す。
5. **ラン型を忘れない**: 失敗=ラン終了なので、中盤以降の 1 ステージの
   クリア率を欲張って下げると Lv15 到達者がいなくなる(§7 の累積計算参照)。

### 4.3 ゲームが説明しない「4 つの暗黙ルール」(実装依存事項)

プレイテストレビューで、config 調整だけでは解決できない伝達課題が 4 つ
特定された。**本設計のクリア率目標はこれらの手当てを前提とする。**
未対応のままだと、カジュアル層は「理由の分からない失敗」を重ねて離脱する。

| #   | 暗黙ルール                                              | 必要な手当て(案)                                                                                 |
| --- | ------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 1   | 異色 2 匹は失敗(−20)                                    | 失敗時に理由を表示する(例:「ちがう色 2 匹はダメ!」)。現状の "Bad Loop!" だけでは理由が伝わらない |
| 2   | 1 匹囲み=色替え                                         | 色替え発動時のエフェクト強化と、Lv4 開始時のヒント表示                                           |
| 3   | large は線を遠くから感知して逃げる                      | Lv1 で安全に体験させる(§5.2 Lv1)+ large が線を警戒した瞬間の視覚フィードバック(ビクッと震える等) |
| 4   | スペシャル蝶を捕る前に needCount 到達するとボーナス消滅 | ボーナスを逃したリザルトで「光る蝶を捕まえてからクリアするとボーナスステージ!」と明示する        |

これらは GitHub Issue として個別に切り出すことを推奨する。

### 5.2 各ステージの特徴と想定戦略

#### Lv1「はじめてのループ」— 操作を覚える

-   **特徴**: 遅い large 蝶が 8 匹、2 色のみ。お助け花なし。要求 6 匹はペア捕り 3 回で足りる。
-   **想定戦略**: 近くにいる同色 2 匹を見つけてゆっくり囲む。それだけで勝てる。
-   **学習目標**: ループを閉じる操作、「同色なら捕れる」ルール、そして
    **「蝶のそばに線を引くと逃げる」こと**。
-   **設計意図**: large は遅く初心者向きだが、線感知半径が最大(14)なので、
    蝶に触れるように線を引けば必ず「逃げる」挙動を安全に体験できる。
    この体験を Lv1 で必ず 1 回はさせておくことが、Lv6 を理不尽にしないための
    布石になる(§4.3-3 の視覚フィードバックと併せて機能する)。
    失敗しても −20 点が軽い傷で済む蝶数にしてある。

#### Lv2「ちいさな群れ」— 数は正義

-   **特徴**: 最速の small が 16 匹と大量にいる。2 色なので画面は同色だらけ。
-   **想定戦略**: 速くて 1 匹ずつは追えないが、16 匹もいれば**どこを囲っても
    同色が 2〜3 匹入る**。雑に囲ってもクリアできる。
-   **学習目標**: 「数が多ければまとめて捕れる」こと。お助け花(初登場)を取ること。
-   **設計意図**: small の速さを「数の多さ」で相殺し、Lv1 と逆の性質を体験させて
    サイズごとに手触りが違うことを刷り込む。「追わずに待ち構える」高度な動きは
    ここでは要求せず、Lv8 の再演時の学びに回す。

#### Lv3「三色の庭」— コンプ捕りの解禁

-   **特徴**: 3 色になり、同色ペアが見つけにくくなる。
-   **想定戦略**: 同色捕りに加え、**3 色 1 匹ずつの「コンプ捕り」(×20 点)** が使える。
    異色 2 匹は失敗(−20)なので、「2 匹の混色だけは絶対に囲まない」判断を学ぶ。
-   **学習目標**: コンプ捕りとバッドループの境界線。
-   **設計意図**: このステージが機能するかは §4.3-1 の「失敗理由の表示」に
    強く依存する。得点差だけからのルール推測は期待しない。

#### Lv4「いろがわりの午後」— 単騎ループ=色替え

-   **特徴**: 色替え蝶が初登場。羽の中央にサブ色が見えるが、慣れないと色を誤認する。
-   **想定戦略**: 邪魔な位置の 1 匹をわざと単騎ループして**色を変え、同色群を製造する**。
    「囲えなかった失敗」が「色替えという成果」に変わるため、実は救済でもある。
-   **学習目標**: 単騎ループの能動的活用。
-   **設計意図**: need を Lv3 から**据え置き(10)**、数字に追われず色替えで
    遊ぶ余白を作る。1 匹囲みは通常「避けるべき行動」として学習されているため、
    自力発見の導線が細い — §4.3-2 のヒント表示を前提とする。
    色数 3 のままこの 1 点だけを教える。

#### Lv5「★ひかりの予感」— 初めてのスペシャル蝶

-   **特徴**: 開始 10 秒で金色に光る蝶が 1 匹だけ出現。倍率蝶(x2)も低頻度で混ざり始める。
    サイズは random で、画面が初めて「にぎやか」になる。
-   **想定戦略**: スペシャル蝶を見つけたら**同色の仲間と一緒に囲んで確保**
    (単騎では色替え扱いになり捕れない)。そのあと通常蝶でクリアラインへ。
-   **学習目標**: スペシャル蝶の存在と「クリアも確保も両方要る」こと。倍率蝶の見た目。
-   **設計意図**: need 12 と軽めに据え、認知負荷の増加分を相殺する。
    **初見プレイヤーは高確率で「先にクリアしてしまい」ボーナスを逃す**。
    これは仕様上避けられないため、逃した直後のリザルトで種明かしをして
    (§4.3-4)、「次の 5 の倍数ステージで狙おう」という動機に変換する。
    初回は逃す前提の設計であり、Lv10 で回収できれば良い。

#### Lv6「おおものラッシュ」— 大きい蝶は大きく囲め

-   **特徴**: large × 色替えの組み合わせ。蝶数 10 とやや少なく、線感知半径 14 が牙をむく。
-   **想定戦略**: 小さいループを高速で回す戦い方が通用しない。**先回りして大きな円を
    先に描き始め**、蝶が線に気づく前に閉じる。色替えで同色 3 匹を作ってから
    大きく一網打尽が理想形。
-   **学習目標**: サイズに応じてループの大きさと引き始め位置を変えること。
-   **設計意図**: Lv1 で体験済みの「線を感知して逃げる」挙動が本格的に立ちはだかる、
    フェーズ 2 の山その 1。Lv1 での事前体験と §4.3-3 の視覚フィードバックが
    ない場合、このステージは「当たり判定がおかしい」と誤解されるリスクが高い。

#### Lv7「四色の乱舞」— 色の経済学

-   **特徴**: 4 色。同色ペアの自然発生率が体感で急落する。色替えなしなので
    盤面の色構成は固定。
-   **想定戦略**: 同色を待つより**バラバラ 3 匹 or 4 匹のコンプ捕り**が主力になる。
    4 色コンプ(80 点)は成立が難しいので、現実解は 3 色コンプの連発。
-   **学習目標**: 「同色狙い」から「コンプ狙い」への戦略スイッチ。
-   **設計意図**: コンプ捕りを主力戦術として一度確立させる。フェーズ 2 の山その 2
    なので need は 14 に据え置く。直後の Lv8 は色数を 3 に戻して爽快枠。

#### Lv8「ちいさな嵐」— 速さと数の応用試験

-   **特徴**: 高速 small 16 匹+色替え。x3 倍率蝶が初登場。花 3 個。
-   **想定戦略**: 追いかけるのではなく、群れが自然に重なる場所に**先に線を引いて
    待ち構える**(Lv2 の物量戦の上位互換)。x3 蝶が同色群にいたら最優先で囲む
    (4 匹同色 + x3 = 120 点)。freeze / gather を引けたら大量捕獲チャンス。
-   **学習目標**: 「待ち構え」の技術。倍率蝶を「狙って」組み込むスコアメイク。
-   **設計意図**: need 16 からまとめ捕りが必要になる(平均 1.6 匹/ループ)。
    その要求を蝶数 16 と花 3 で支える。花に頼らなくても届く水準。

#### Lv9「五色の楽園」— 色の飽和

-   **特徴**: 全 5 色が出揃う。同色ペアは滅多に並ばない。サイズも random で視認難度最大級。
-   **想定戦略**: **コンプ捕り(3〜4 色)が主力の安定打**。Lv7 で確立した技術が
    そのまま解答になる。5 匹全色コンプ(100 点)はロマン枠。gather 花を引けたら
    色ごとの島ができ、同色まとめ捕りが「ボーナスタイム」として解禁される。
-   **学習目標**: 色が増えるほどコンプ捕りの価値が上がる、という経済の理解。
-   **設計意図**: need は 16 に据え置き、色の難しさだけで勝負するフェーズ 3 の山。
    クリアラインはコンプ捕り(3 匹 × 6 ループ)だけで届く計算にしてあり、
    **gather はあくまで加速装置**。運が悪くても腕で越えられる。

#### Lv10「★収穫祭」— 撒いて、集めて、刈り取る

-   **特徴**: 3 色に戻り倍率蝶が 20% と大盤振る舞い。花 4 個。スペシャル蝶あり。
-   **想定戦略**: 色数が少なく蝶数 16 なので同色群が常時できる。倍率蝶入りの
    同色 5 匹(x3 なら 150 点)の**収穫コンボ**が決まりやすい。Lv5 でボーナスを
    逃したプレイヤーの**リベンジ戦**であり、今度は「スペシャル蝶を先に確保 →
    クリア」の手順を試せる。「クリア直前で止めて、最後のループで大量捕獲 →
    超過ボーナス 100 点/匹」という上級テクの練習場としても最適。
-   **学習目標**: スコアを「作る」意識。超過ボーナスの存在。
-   **設計意図**: フェーズ 3 の谷であり祝祭。気持ちよく稼がせてボーナスステージに送る。

#### Lv11「いたずらな羽色」— 情報戦

-   **特徴**: random サイズ × 4 色 × 色替えの三重奏。画面の情報量が最大になる。
-   **想定戦略**: 主力は Lv7 と同じ 3 色コンプ捕り。加えて色替えを使えるなら
    「あの子を単騎で替えれば 3 匹同色」を計画的に作れる(上級者はここで差がつく)。
-   **学習目標**: サブ色の先読み。ここまでの全読解スキルの中間試験。
-   **設計意図**: need 20 は平均 2.0 匹/ループ。色替えなしでもコンプ捕り連発で
    届くが、色替えを使いこなすと余裕が生まれる — 「必須」ではなく「差がつく」
    位置に置くのが肝。上級者判定ステージ。

#### Lv12「群れの王国」— 最大の爽快枠

-   **特徴**: small 20 匹・2 色のみ。画面が蝶で埋まる。倍率 20% / x3。花 4 個。
-   **想定戦略**: どこを囲っても同色群がいる。**1 ループ 5 匹以上**を狙い続ける
    物量戦。ただし 2 色なのでコンプ捕りは存在せず、混色事故(異色 2 匹)だけが敵。
-   **学習目標**: 大量捕獲の快感と、雑に囲む誘惑への抵抗。
-   **設計意図**: need 22 と数字は重いが、成立手段を過剰供給して
    「難しいのに気持ちいい」を作る。Lv11 の頭脳戦との緩急。

#### Lv13「五色乱舞」— 色マスタリー最終試験

-   **特徴**: 5 色 × 色替え × 蝶数 14。同色ペアの自然発生に頼れない。
-   **想定戦略**: 3〜4 色コンプ捕りを主軸に、単騎色替えでコンプ素材や同色ペアを
    「調達」する。×20 の単価が高いので、コンプが回れば得点効率はむしろ良い。
-   **学習目標**: コンプ捕りと色替えの完全な統合。
-   **設計意図**: 本作の「色システム」の集大成にして最難関。need 24 は
    3 匹コンプ × 8 ループ相当で、**色替えを使わなくても理論上届く**が
    失敗の許容回数がほぼない。色替えで素材を作れる人は余裕を持って越えられる。
    蝶数を 14 に増やしてコンプ素材の枯渇を防ぎ、花 4 個を生命線として配る。

#### Lv14「おおものの四色」— 精密ループ最終試験

-   **特徴**: 線感知の早い large が 4 色で 13 匹。色替えあり。
-   **想定戦略**: 大きなループを先読みで引く Lv6 の技術+色管理。freeze 花の
    価値が最大(large の遅さが完全に無力化される)。
-   **学習目標**: ループ精度の集大成。
-   **設計意図**: 「色の 13」に対する「操作の 14」。二本柱の最終試験を分けることで、
    プレイヤーが落ちたとき自分の弱点(色判断か、ループ精度か)を認識できる。
    large × 多色は成立コストが最も高い組み合わせのため、need は 13 から
    **据え置き(24)**、蝶数 13 と花 4 で下支えし、13 と同格の難度に収める
    (13 より難しくなってフィナーレ前に折られるのが最悪のシナリオ)。

#### Lv15「★グランドフィナーレ」— 全部載せの祝祭

-   **特徴**: random サイズ・4 色・色替え・倍率 20%/x3・花 5 個・スペシャル蝶。
    全種別が一画面に共存する。
-   **想定戦略**: これまでの全戦術の自由演技。花が 5 個降るため常に何かが起きる。
    スペシャル蝶を確保してボーナスステージで有終の美を飾るのが理想の結末。
-   **設計意図**: 難度は 13/14 より明確に下げる(need 据え置き 24、蝶数 16 で
    成立手段が多い)。「試験の後の卒業式」であり、ここまで来たプレイヤーの
    大半が気持ちよく終われるクリア率(§7 で 70%)を目標にする。

### 5.3 レベル 16 以降(エンドレス帯)

既存の自動生成(群れ/カラフル/大物/混沌の 4 パターン巡回、need +2/レベル)を
継続する。§3 の見積もりから need 30 超(Lv18 前後)で急速に達成困難になり、
**Lv20 前後が事実上の到達限界**となる。

これは欠陥ではなく、**「限界が来ることが確定しているエンディング型」の
エンドレスモード**として明確に割り切る。need の伸びは腕でも覆せないため、
やり込みの軸は「どこまで行けるか」単独ではなく**「ラン合計スコア」との複合**に
置く(同じ Lv18 で終わっても、道中の稼ぎで差がつく)。その上で:

-   生成パターンの `helpObjectNum` 下限を 3 に引き上げる(need の伸びに対する救済)。
-   Lv20 のスペシャル蝶+ボーナスステージを「実質エンディングの花火」と位置づける。
-   将来案として「エンドレス帯のみ超過捕獲分を次ステージの need に充当する」
    緩和があり得るが、コード変更を要するため本書のスコープ外の検討事項とする。

## 7. 検証計画

数値は仮説であり、プレイテストで以下を計測して調整する。
検証は `stage-config-debug.json` に §5.1 の値を写して `npm run debug` で行う。

### 7.1 最優先: 背骨の実測

**有効ループ数(60 秒あたりの意味のあるループ回数)を初見層/熟練層で実測する。**
§3 の仮説(初見 8〜10 / 熟練 12〜15)が楽観側にズレていた場合、
全ステージの needCount を一括で −2 補正するフォールバックを用意しておく。

### 7.2 ステージ別クリア率

ラン型(失敗=ラン終了)なので、クリア率は直列に掛かる。目標:

| 帯      | ステージ別クリア率目標 | 累積到達率(目安) |
| ------- | ---------------------- | ---------------- |
| Lv1-3   | 95%+                   | Lv4 到達 86%     |
| Lv4-7   | 85%                    | Lv8 到達 45%     |
| Lv8-11  | 75%                    | Lv12 到達 14%    |
| Lv12-14 | 65%                    | Lv15 到達 4%     |
| Lv15    | 70%                    | 完走 3%          |

累積 3% は「1 ランで一気に完走する確率」であり、リトライを重ねて上達する
前提の数字である(中盤以降の到達者は上達による自己選抜がかかるため、
実測のステージ別クリア率は表より高く出るはず)。実測がこの目安を大きく
下回る帯は、needCount か成立手段(蝶数・花)を調整する。

### 7.3 その他の計測指標

1. **バッドループ率**(失敗ループ/全ループ): 20% 超のステージは色数か蝶数を見直す。
   §4.3-1(失敗理由表示)の実装前後で比較できるとなお良い。
2. **gather 有無別のクリア率**: 差が 15pt を超えるステージは「gather 引けたか
   どうかの運ゲー」になっている兆候。needCount を下げるか蝶数を増やす。
3. **1 ループ平均捕獲数**: §3 の必要平均を実測が下回るステージは成立手段を追加。
4. **お助け花の取得率**: 50% を大きく下回る場合、花の落下位置・速度の調整を検討
   (本書のスコープ外だがボトルネックになり得る)。
5. **Lv5 でのボーナス取り逃し率**: 高くて構わない(初回は逃す前提)が、
   Lv10 での回収率が低い場合は §4.3-4 の伝達が機能していない。
